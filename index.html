RPO → Agora migration: insurance states

Migrating the RPO (invited/referred patient) onboarding experience into the unified Agora checkout. Most insurance states already work in Agora today via shared components. This doc focuses on the gap — the RPO-specific states that need to be built into the unified InsuranceStep.

Where things stand: Jesse Dawson's draft PR #54774 built the unified checkout scaffold with entry points, registry-based step sequencing, and a DeferredCheckoutEntry that detects isDeferredVerification from the provider event channel. The InsuranceStep component has display-state enums for RECONCILE_INSURANCES and PENDING_INSURANCE, but the actual card UI is placeholder text.

Gap: Build the reconciliation + pending confirmation card UI, port useInsuranceAndContactInfoQuery into the unified flow's adapter, and design + build the self-pay CONFIRM_BILLING_METHOD step.
